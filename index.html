Fluid Webpage

What is a Fluid Webpage?

A fluid webpage is our first step in making our fixed website, responsive. It is an amazing step up from the previous fixed website. As we change screen sizes our webpage will scale, thus it will change size proportionally with the screensize. Rather than just getting cut off, when the screensize is changed. This will allow the user to interact more comfortably while viewing our webpage despite what device they are using.
Pros: realitively easy to create and will scale with different medias.
Cons: will not automatically adjust to standard screen sizes, will not use media queries, which we will get into later. Let's jump in!

Web Types

Before we dive into the math we should review the different types of webpages.

  • Fixed: This is your clasic webpage. No scalability.
  • Fluid: This webpage does scale with different screen resolutions. However, without media queries this is only so powerful.
  • Adaptive: An adaptive webpage utalizes media queries. They are resolution driven, thus for specific devices we can have a perfectly designed website.
  • Responsive: Adaptive based on relative widths, this time with content driven media queries. Allowing for perfect website viewing regardless of device screen size or resolution.

Convert the font!

We are finally about to get into some math. Have no fear, it is pretty basic and I am sure you will have no issues. We are going to convert the font form pixels to ems

  • Convert font size from px ---> em
  • To do this we get the target pixel, the item we want to resize, and devide it by the context pixel size, the default size of the font
  • Target(px) / Context(px) = Results(em)
    24px(target) / 16px(context and default) = 1.500em
  • If you are targeting a nested font, the font-size it is nested in, becomes the context.
  • Based on the last equation if we take a size 12px font nexted inside a 16px font
  • 12px / 16px = 0.75em

Working with Percentages

Great now our font is scalable, but what about the pixel settings for widths and margins. We will start by making the container the correct proportional size.

  • Convert container from px ---> percentages
  • We are going to start with the container, remember this was set to 1000px width
  • Set Max-Width: 1000px;
  • This will tell the webpage that the max possible width is 1000px
  • Then we want to set the container's actual wdith to 90%
  • Width: 90%;
  • Now that the main container is scaled lets move onto the grid widths
  • Make the final grid 100%, then for all proceeding grids divide its sizepx(grid size) by 1000px(total size) * 100
  • grid_11 915px / 1000px = .915 * 100 = 91.5%

Convert the Margins

Awesome job, we now have the font-size in ems and the container div in percentages. Now we need to make sure the margins scale.

  • Convert margins from px ---> percentages
  • In order to do this conversion, we take the margin px and divide them by the containing divs px
  • 10px(margin) / 500px(containingdiv) = .02 * 100 = 2%
  • Remember this is based off of the containing div
  • With margins set we can now fix our final issue, which is images
  • Set images to have a max-width of 100% in the stylesheet and we will have scalable images
  • img {
    max-width:100%;
    }

Adaptive Webpage

Time to begin making an Adaptive Webpage

Wow, the progress we have made is amazing, but we are not through yet. A fluid webpage is great and will scale with different screensizes, however it will not adapt to these different resolutions. Meaning, whether we view the webpage on a mobile device, a tablet, or a PC it will always look the same. Most of the time as we scale to smaller devices we would like to have less viewable content but have that content be more specific. We are going to achieve this by using media queries. Media queries are the huge step that is going to allow us to make self adapting webpage. Once mastered we will only be steps away from completing a responsive webpage.

Media Queries

Media queries are the most powerful tools when dealing with adaptive webpages. It allows for the webpage to "adapt" to different screensizes and resolutions making a website as user friendly as possible. Generally when dealing with media queries we want to make the most important imformation easily accessible on smaller screen devices.

So what does the user want out of our website? When shrinking our fluid webpage are aspects overlapping? Are the important functions easily navigated?

These questions and more are going to be what influences the content on your website. We have already worked with media queries, look at the head document. It is hard to explain how to make a website adaptive, so we will show you some sample code.

@media screen and (max-width: 480px) {

.grid_1,
.grid_2,
.grid_3,
.grid_4,
.grid_5,
.grid_6,
.grid_7,
.grid_8,
.grid_9,
.grid_10 {
width:100%;
}
}

Let's dive into the code

So, what is that partially familure and partially alien css code doing? the @media is alererting the computer media from "screen" to type of screen we are looking at. Distinguishing the screen based on the width of the monitor. We used a max-width calculation which will take into account our changes listed, up until a screen reaches a size greater than 480px in width. So with multiple media queries we can set up different layouts for different screen widths. The code to the left(or above), is what I used to make this website which has 2 columns to only have 1 when viewed on a smaller device.
Great, we have media queries for different screensizes by using Google Window Resizer and implementing small edits. However, you still need to reference this piece of css in your html. You do this by using a viewport meta tag. Add the following meta tag to the head of your document like the media tags.

< meta name="viewport" content="width=device-width, initial-scale =1.0, user-scalable=no >

Break Point

Well we are here, a webpage that is adaptive to certain website sizes. However there are many more screen resolutions we are not tailoring to. So we require a change of mind, rather than creating a website that fits to certain devices, we need to create a website that looks perfect regardless of what screensize it is used on. This sounds much more difficult than it is. This new mentality allows us to be selfish, and decide how all screens will view our website. We do this by starting with the smallest width, probably somewhere around 480px, in the browser with, Windows Resizer, scale your website. Then begin expanding your website until the content looks a little weird. That is our break point, record that width. Continue expaning your websites and marking the break points as you proceed.

The Fix

Great we have a few break points, hopefully they are close to the widths we already established. Let's edit our media queries widths to the break point widths. Add any aditional media queries if neccessary. Now just like when making an adaptive website, edit the .css to prettify the webpage.
